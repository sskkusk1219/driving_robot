# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**driving_robot** - シャシダイナモ向け自動運転ロボットシステム

### プロダクトコンセプト
- **高精度な車速トレース**: フィードフォワード＋フィードバック制御により、基準車速に対して±0.2km/h以内の精度で追従
- **人の運転からの脱却**: WLTPなどの運転サイクルをロボットが代替し、疲労・ばらつきのない長時間走行を実現
- **シンプルで拡張性の高いシステム**: 直感的なWeb GUIと充実したAPIにより、初心者から上級エンジニアまで対応

### プロダクトビジョン
シャシダイナモ試験における人的要因によるばらつきをなくし、誰でも・いつでも・再現性の高い走行試験を実現する。
アクチュエータ制御・運転モデル学習・リアルタイムモニタリングを統合したシステムにより、実験エンジニアが運転に費やしていた時間をデータ分析や実験設計に充てられるようにする。
またAPIと拡張インターフェースを提供することで、シャシダイナモ・計測器との連携を容易にし、実験手法開発エンジニアの研究活動を加速する。

### 目的
- WLTPなどの運転サイクルを再現性高く自動走行する
- 10時間以上の連続走行を無人で実現する
- 車両ごとの運転モデルを学習・蓄積し、精度を継続的に改善する
- シャシダイナモ・計測器との外部連携を可能にするAPIを提供する

---

## ターゲットユーザー

### ペルソナ1: 実験エンジニア（プライマリー）
- **年齢**: 25〜50歳
- **職業**: 自動車メーカーや研究機関の実験エンジニア
- **経験**: シャシダイナモでの走行試験経験あり、運転ロボットの操作は初めて
- **現在の課題**:
  - 毎日同じ運転サイクルを繰り返すことによる疲労とばらつき
  - 運転ロボットの操作が複雑そうで導入に不安
  - 運転に時間を取られ、実験準備やデータ分析が後回しになる
- **期待する解決策**:
  - 説明書なしでも走行開始できるシンプルなGUI
  - 正確なアクセル・ブレーキ制御
  - 走行中のリアルタイムモニタリングとログ取得
- **典型的なワークフロー**:
  1. 車両をシャシダイナモにセット
  2. PCのブラウザでGUIを開く
  3. 車両プロファイルを選択・キャリブレーション実施
  4. 走行モードを選択して実行
  5. モニタリングしながらデータ分析の準備

### ペルソナ2: 実験手法開発エンジニア（セカンダリー）
- **年齢**: 25〜50歳
- **職業**: 実験手法を開発するエンジニア
- **経験**: 機械設計・制御・プログラミングの有識者
- **現在の課題**:
  - 運転ロボットのカスタマイズ性が低く実験手法の開発に制約がある
  - 運転ロボット・シャシダイナモ・計測器の連携に手間がかかる
- **期待する解決策**:
  - 外部システムから制御できるAPI
  - リアルタイムモニタリングと柔軟な制御インターフェース
  - 充実したドキュメント

## 成功指標(KPI)

### プライマリーKPI（Must）
- **車速追従精度**: 基準車速に対して実車速が±0.2km/h以内でトレースできること
- **オーバー/アンダーシュート**: 基準車速に対してオーバー/アンダーシュートを繰り返さない運転をすること
- **連続走行時間**: 10時間以上の連続走行が可能なこと
- **操作習得時間**: 走行開始まで説明書なしでGUI操作が完結できること
- **走行モード**: WLTP・NEDC・UDDS・AMAなど複数の運転サイクルを再生できること

### セカンダリーKPI（Should）
- **カスタマイズ性**: GUIからロボットの動作を細かく制御できること
- **API提供**: 外部システムからロボットを制御・データ取得できるAPIが提供されていること
- **ログ保持**: 過去3ヶ月分のログをGUIから確認できること

## 機能要件

### コア機能（MVP）

#### 1. 車両プロファイル管理

**ユーザーストーリー**:
実験エンジニアとして、複数の車両を扱うために、車両ごとのキャリブレーションデータと運転モデルをプロファイルとして保存・切り替えたい

**受け入れ条件**:
- [ ] プロファイル名を入力して新規作成・保存できる
- [ ] 作成済みプロファイルを一覧から選択して読み込める
- [ ] PIDゲイン・最大開度・最高車速・停止処理設定をプロファイルに保存できる
- [ ] キャリブレーションデータ（min/max位置）がプロファイルに紐づく
- [ ] 運転モデルがプロファイルに紐づく

**優先度**: P0（必須）

---

#### 2. キャリブレーション

**ユーザーストーリー**:
実験エンジニアとして、正確なペダル制御のために、アクセル・ブレーキのゼロフル位置を自動で検出したい

**受け入れ条件**:
- [ ] アクセル・ブレーキともに電流急増検出によるゼロフルを自動実施できる
- [ ] キャリブレーション中は最大開度設定を無視して全ストロークを検出できる
- [ ] バリデーション（接触点検出・フル検出・ストローク妥当性・順序）でNGの場合GUIにエラーを表示して停止する
- [ ] 成功したキャリブレーションデータをプロファイルに保存できる

**優先度**: P0（必須）

---

#### 3. 学習運転

**ユーザーストーリー**:
実験エンジニアとして、車両の運転特性を取得するために、学習専用の運転パターンで自動走行してログを収集したい

**受け入れ条件**:
- [ ] 走行前チェック（通信・サーボ・キャリブレーション・プロファイル・UPS残量・位置）をすべてパスした場合のみ開始できる
- [ ] 車両プロファイルの最大開度・G上限を超えるパターンは自動スキップされる
- [ ] 実車速・アクセル開度・ブレーキ開度・位置・電流値を100ms周期でログ保存できる
- [ ] 20分以内で完了する

**優先度**: P0（必須）

---

#### 4. 運転モデル学習

**ユーザーストーリー**:
実験エンジニアとして、精度の高い自動走行のために、収集したログから車両ごとの運転モデルを学習したい

**受け入れ条件**:
- [ ] 学習運転ログまたは過去の走行ログからモデルを学習できる
- [ ] 学習したモデルが車両プロファイルに紐づけて保存される
- [ ] 再学習によってモデルを更新できる

**優先度**: P0（必須）

---

#### 5. 走行モード管理

**ユーザーストーリー**:
実験エンジニアとして、様々な運転サイクルを実行するために、基準車速CSVをアップロードして管理したい

**受け入れ条件**:
- [ ] 基準車速CSVをアップロードしてモード名を付けて保存できる
- [ ] 走行モード一覧から選択して実行できる

**優先度**: P0（必須）

---

#### 6. 自動運転（走行モード）

**ユーザーストーリー**:
実験エンジニアとして、再現性の高い試験のために、基準車速CSVに従ってアクセル・ブレーキを自動制御したい

**受け入れ条件**:
- [ ] 走行前チェックをすべてパスした場合のみ開始できる
- [ ] 運転モデルと基準車速からフィードフォワード制御用目標位置を10ms周期で算出できる
- [ ] フィードフォワード＋PIDフィードバック制御で実車速を基準車速に追従させられる
- [ ] 基準車速・実車速・アクセル開度・ブレーキ開度をリアルタイムでGUIに表示できる
- [ ] 設定した追従逸脱条件（例：±2.0km/h・4.0s以上）で自動停止できる
- [ ] 走行ログを100ms周期で保存できる
- [ ] 非常停止スイッチ押下で全アクチュエータが即座に原点復帰する

**優先度**: P0（必須）

---

#### 7. 安全システム

**ユーザーストーリー**:
実験エンジニアとして、万が一の際にも安全に停止できるように、非常停止・過電流検知・AC電源断対応が欲しい

**受け入れ条件**:
- [ ] 非常停止スイッチ（室内・操作エリアの2箇所）のいずれかを押すと全アクチュエータが即座に原点復帰する
- [ ] 電流値が閾値を超えた場合に自動停止する
- [ ] AC電源断検知時に安全停止→ログ保存→PostgreSQL正常終了→シャットダウンの順に処理される
- [ ] 5V側UPS（X1201）のバッテリー残量をGUIに表示できる

**優先度**: P0（必須）

---

#### 8. 手動操作

**ユーザーストーリー**:
実験エンジニアとして、アクチュエータの動作確認や特定の開度での試験のために、GUIからペダルを手動で操作したい

**受け入れ条件**:
- [ ] 走行前チェックをすべてパスした場合のみ開始できる
- [ ] GUIのスライダーまたは入力フォームでアクセル・ブレーキ開度をリアルタイムで調整できる
- [ ] 操作中のステータスをリアルタイムで確認できる
- [ ] 終了後にログが保存される

**優先度**: P0（必須）

---

#### 9. ログ管理・確認

**ユーザーストーリー**:
実験エンジニアとして、試験結果を分析するために、走行ログをGUIで確認・CSVダウンロードしたい

**受け入れ条件**:
- [ ] 過去3ヶ月分のログをGUIから一覧・グラフ表示できる
- [ ] ログをCSV形式でダウンロードできる
- [ ] 3ヶ月経過後のログはCSV+圧縮して外付けUSB SSDに自動アーカイブされる
- [ ] アーカイブ先がストレージの80%超で古いものから自動削除される

**優先度**: P0（必須）

---

#### 10. 起動・初期化

**ユーザーストーリー**:
実験エンジニアとして、安全に作業を開始するために、システム起動時に通信確認と初期化を実施したい

**受け入れ条件**:
- [ ] 電源ONでWeb GUIが自動起動（待機状態）する
- [ ] 起動時に通信（ttyUSB0・ttyUSB1・CAN）を自動確認し、NGはGUIにエラー表示する
- [ ] 「初期化」ボタン押下でアラームリセット→サーボONを実行する
- [ ] 前回正常終了なら原点復帰をスキップ、異常終了・初回なら原点復帰を実施する

**優先度**: P0（必須）

---

### 拡張機能（Post-MVP）

#### 11. タイムスケジュール

**ユーザーストーリー**:
実験手法開発エンジニアとして、基準車速を使わずに時系列でペダル動作を自動化するために、タイムスケジュールを作成・実行したい

**受け入れ条件**:
- [ ] 時刻とアクセル・ブレーキ開度を表形式で入力できる
- [ ] ループ設定ができる
- [ ] スケジュール名を付けて保存・一覧管理できる

**優先度**: P1（重要）

---

#### 12. シーケンス

**ユーザーストーリー**:
実験手法開発エンジニアとして、複合的な試験を一括実行するために、走行モードとタイムスケジュールを任意の順序で組み合わせたシーケンスを作成・実行したい

**受け入れ条件**:
- [ ] 走行モードとタイムスケジュールを任意の順序で組み合わせられる
- [ ] シーケンス名を付けて保存・一覧管理できる

**優先度**: P1（重要）

---

#### 13. 外部API・TCP/IP連携

**ユーザーストーリー**:
実験手法開発エンジニアとして、外部システムとの連携のために、ロボットを制御・データ取得できるAPIが欲しい

**受け入れ条件**:
- [ ] REST APIで走行開始・停止・状態取得ができる
- [ ] TCP/IPでリアルタイム値（車速・開度など）を取得できる
- [ ] APIドキュメントが提供されている

**優先度**: P1（重要）

---

#### 14. シグナルタワー

**ユーザーストーリー**:
実験エンジニアとして、システム状態を遠くからでも把握するために、シグナルタワーで状態を確認したい

**受け入れ条件**:
- [ ] 待機中（緑）・走行中（青）・アラーム（赤）・非常停止（赤点滅）を表示できる

**優先度**: P2（できれば）

---

#### 15. オンライン学習

**ユーザーストーリー**:
実験手法開発エンジニアとして、走行精度を動的に向上させるために、走行しながら運転モデルを更新したい

**優先度**: P2（できれば）

## 非機能要件

### パフォーマンス
- **制御周期**: アクチュエータへの位置指令を10ms周期で更新できること
- **ログ記録**: 100ms周期で走行データを記録できること
- **GUI更新**: リアルタイムモニタリングを100ms以下の遅延で表示できること
- **起動時間**: 電源ONからGUI操作可能状態まで60秒以内

### 信頼性
- **連続稼働**: 10時間以上の連続走行が可能なこと
- **データ保全**: ログのPostgreSQL書き込みが正常終了すること
- **異常停止時の安全**: エラー・非常停止・AC断いずれの場合もアクチュエータが原点復帰すること
- **UPS対応**: AC電源断時に24V・5V両系統のUPSで安全停止シーケンスを実行できること

### ユーザビリティ
- **操作習得**: 説明書なしで走行開始まで操作できること
- **アクセス方法**: ブラウザからIPアドレスを入力するだけで接続できること（アプリインストール不要）
- **エラー通知**: エラー発生時に原因をGUIに表示すること

### セキュリティ
- **ローカルネットワーク限定**: 同一LAN内からのみアクセス可能とする
- **機密情報の非保存**: IDやパスワードなどの機密情報を保存しないこと

### スケーラビリティ
- **ログ容量**: アクティブDBを3ヶ月・アーカイブを外付けSSDで管理し、80%超で自動削除
- **車両プロファイル数**: 複数車両のプロファイルを無制限に管理できること

## スコープ外

明示的にスコープ外とする項目:
- マルチユーザー認証・権限管理
- クラウド連携・リモート（社外）アクセス
- 24V側UPS以外の電源バックアップ設計（機種選定はTBD）
- 車両のOBD2・CANバスからの車速取得（シャシダイナモCANのみ対応）
- Windows/macOS対応（Raspberry Pi Linux専用）
- グローバル環境へのライブラリインストール
